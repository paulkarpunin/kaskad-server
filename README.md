# gokaskad: Интеллектуальный NAT-маршрутизатор

**gokaskad-server** — это автоматизированный инструментарий инфраструктурного уровня для развертывания отказоустойчивых сетевых каскадов (NAT-мостов) на базе ОС Linux. 

Система предназначена для инкапсуляции и перенаправления клиентского трафика на конечный узел. Это позволяет эффективно скрывать реальную точку назначения (Endpoint). Проект объединяет в себе ядро конфигурации `iptables`, модуль пассивной телеметрии и интерактивный Telegram-демон (Command & Control).



## 🏛 Архитектура и Технологии

Проект реализован по паттерну **Multi-call binary** (многофункциональный исполняемый файл) и опирается на 4 фундаментальных уровня:

1. **Core (Маршрутизация):** Детерминированное управление правилами ядра (PREROUTING / POSTROUTING / FORWARD) с использованием криптографических меток (Tagging). Скрипт управляет логическими туннелями, а не разрозненными правилами, полностью исключая риск «гонки данных» (Race Conditions) через атомарные транзакции `iptables-save`/`iptables-restore`.
2. **Watchdog (Телеметрия):** Автономный анализатор задержек сети (ICMP/TCP Ping). Использует алгоритм конечного автомата (State Machine) для защиты от ложных срабатываний при сетевом джиттере.
3. **C2 (Command & Control):** Интерактивный Telegram-бот, работающий как системный `systemd` демон (Long Polling). Обеспечивает защищенный канал управления сервером.
4. **Оптимизация ядра:** Автоматическая конфигурация параметров `sysctl` (алгоритм контроля перегрузок **Google BBR** и очередь `fq`) для минимизации потерь пакетов и снижения latency.

## 🚀 Быстрый старт (Установка)

Для развертывания системы на чистом сервере выполните следующую команду. Конвейер автоматически загрузит последнюю версию из ветки `main`, установит зависимости (включая `jq` и `netfilter-persistent`) и запустит главное меню конфигуратора:
```
wget -O install.sh https://raw.githubusercontent.com/paulkarpunin/kaskad-server/main/install.sh && chmod +x install.sh && ./install.sh
```
или

```
bash <(curl -Ls https://raw.githubusercontent.com/paulkarpunin/kaskad-server/main/install.sh)
```

## ⚙️ Системные требования:

ОС: Debian 10+ / Ubuntu 22.04+ (дистрибутивы с поддержкой apt и systemd).

Права: Исключительно root (необходимо для модификации ядра и таблиц маршрутизации).

## ⚙️ Эксплуатация и Настройка
При запуске скрипта из терминала (команда gokaskad) открывается интерактивное меню.

### Создание туннелей
Выберите необходимый протокол (UDP для WireGuard/AmneziaWG, TCP для VLESS/XRay) или настройте кастомное проксирование портов. Система запросит целевой IP-адрес и порт, после чего атомарно внедрит правила в ядро и сохранит их в энергонезависимую память.

Внимание: В клиентском VPN-приложении в качестве Endpoint (IP-адреса сервера) необходимо указывать IP-адрес текущего (промежуточного) сервера, а не конечного.

### Telegram-бот (Уведомления и Управление)

#### Скрипт gokaskad поддерживает двунаправленную связь через API Telegram.

- В главном меню перейдите в раздел «Настройки Telegram-бота».

- Введите ваш Bot Token и личный Chat ID.

    *Важно: бот отвечает только на команды, отправленные с вашего авторизованного Chat ID. Запросы от сторонних пользователей аппаратно отбрасываются демоном.*

- Выберите пункт «Запустить службу интерактивного бота».

#### Доступные команды в Telegram:

    /status — Вывод системных метрик реального времени (Uptime, CPU, RAM, Disk, Топ процессов).

    /list — Инвентаризация активных каскадов (чтение меток из базы iptables).

    /delete [ID] — Дистанционное удаление туннеля (например, /delete gokaskad_tcp_443).

    /update — Активация пайплайна Self-Update. Сервер скачивает свежий код с GitHub, валидирует его и бесшовно перезапускает свои службы.

    /backup — Генерация файла резервной копии (Disaster Recovery) и отправка его в диалог.

### Мониторинг (Watchdog)
Вы можете точечно назначить телеметрию для каждого созданного туннеля.
- Перейдите в раздел «Управление мониторингом», выберите туннель, укажите допустимый порог пинга (в миллисекундах) и интервал проверок (в минутах). Если связь с конечным сервером деградирует, Watchdog отправит Telegram-алерт. При нормализации пинга вы получите уведомление о восстановлении сети.


## Аварийное восстановление (Backup & Restore)
В случае блокировки текущего сервера или аппаратного сбоя, система позволяет перенести конфигурацию в 3 шага:

- Выгрузите бэкап через команду /backup в Telegram или через меню сервера.

- На новом сервере выполните базовую установку gokaskad и загрузите файл бэкапа (например, через SFTP).

- Выберите пункт «Резервное копирование -> Восстановить инфраструктуру» и укажите путь к файлу. Скрипт автоматически воссоздаст все туннели, задачи мониторинга и настройки Telegram.

Управление бекапированием и восстановлением также доступно напрямую на сервере, пункт меню "Резервное копирование".

## Безопасная очистка (Системный сброс)
Архитектура скрипта изолирована от остальных подсистем ОС. Если вы выберете функцию «Сбросить ВСЕ настройки», алгоритм проанализирует ядро и удалит только те правила маршрутизации и cron-задачи, которые были созданы маршрутизатором (по префиксу gokaskad_). Сетевые конфигурации сторонних сервисов (Docker, UFW, Fail2Ban) не пострадают.
