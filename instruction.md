# 📚 Техническое руководство по эксплуатации gokaskad-server

## 1. Инициализация и системные требования

Система предназначена для работы на чистых (свежеустановленных) серверах. Скрипт автоматически конфигурирует параметры ядра (`sysctl`), необходимые для оптимальной маршрутизации.

**Команда установки и запуска:**
```
wget -O install.sh https://raw.githubusercontent.com/paulkarpunin/kaskad-server/main/install.sh && chmod +x install.sh && ./install.sh
```
или

```
bash <(curl -Ls https://raw.githubusercontent.com/paulkarpunin/kaskad-server/main/install.sh)
```

## 2. Управление туннелями (Маршрутизация)
    Скрипт управляет трафиком через манипуляцию таблицами iptables (цепочки PREROUTING, POSTROUTING, FORWARD).

### Создание нового каскада (Пункты 1-3 главного меню)
- Выберите протокол, соответствующий вашему конечному VPN-протоколу:

- UDP (Пункт 1): Для WireGuard, AmneziaWG.

- TCP (Пункты 2-3): Для VLESS, XRay, Shadowsocks, MTProto, SSH, RDP.

- Введите IP-адрес конечного сервера, на котором физически установлен VPN.

- Введите Порт (он должен совпадать с портом, который слушает ваш зарубежный VPN-сервер).

**Критически важно: После создания туннеля, в вашем клиентском приложении на телефоне/ПК (например, в приложении Amnezia или v2rayNG) вы должны изменить IP-адрес конечного сервера на IP-адрес текущего промежуточного сервера, где установлен gokaskad.**

### Просмотр и удаление (Пункты 4-6)
- Пункт 4 (Посмотреть активные правила): Выводит список всех туннелей, зарегистрированных в ядре. Каждому туннелю присваивается уникальный ID вида gokaskad_tcp_443.

- Пункт 5 (Удалить одно правило): Атомарно удаляет выбранный каскад из ядра и стирает связанные с ним задачи мониторинга.

- Пункт 6 (Сбросить ВСЕ настройки): Экстренная очистка (Safe Flush). Удаляет всю инфраструктуру gokaskad, не ломая общую сеть сервера и правила других программ.

## 3. Интеграция с Telegram (C2 Демон)
    Для дистанционного управления инфраструктурой предусмотрен модуль Command & Control (C2), работающий как системная служба (systemd).

### Настройка (Пункт 7 главного меню)
- Выберите пункт ```1) Изменить Token и Chat ID.```

- Введите токен вашего Telegram-бота (получить у @BotFather).

- Введите ваш Chat ID (получить у @getmyid_bot). Бот будет реагировать только на команды от этого ID, обеспечивая безопасность управления.

- Выберите пункт ```3) Запустить службу интерактивного бота.```

### Управление через Telegram
- Откройте диалог с вашим ботом и используйте следующие команды:

    ```/status``` — Отчет о состоянии сервера (Uptime, нагрузка на CPU, потребление RAM и диска, таблица ресурсоемких процессов).

    ```/list``` — Вывод списка всех работающих туннелей.

    ```/delete [ID_ТУННЕЛЯ]``` — Удаление конкретного туннеля (например, ```/delete gokaskad_udp_51820```).

    ```/backup``` — Генерация файла аварийного восстановления (отправляется файлом в диалог).

    ```/update``` — Асинхронное обновление скрипта с GitHub.

## 4. Телеметрия и Мониторинг (Watchdog)
    Модуль Watchdog — это пассивный анализатор, который следит за качеством связи между вашим промежуточным и конечным зарубежным сервером.

### Настройка (Пункт 8 главного меню)
Для включения мониторинга у вас должен быть предварительно настроен Telegram-бот.

- Зайдите в меню и выберите ```1) Добавить туннель в мониторинг.```

- Выберите туннель из списка.

- Задайте Порог пинга в мс (например, 150).

- Задайте Интервал в минутах (например, 2).

Логика работы:
Скрипт проверяет доступность узла (ICMP Ping) с заданным интервалом. 

- Если средний пинг превышает порог или узел недоступен, скрипт фиксирует состояние и отправляет алерт ⚠️ ДЕГРАДАЦИЯ СЕТИ. 

- Как только пинг возвращается в норму, вы получаете уведомление ✅ СЕТЬ ВОССТАНОВЛЕНА. 

- Защита от спама (State Machine) гарантирует, что вы получите только одно уведомление на каждое изменение состояния.

## 5. Аварийное восстановление (Disaster Recovery)
Система поддерживает аппаратно-независимое резервное копирование. Это позволяет перенести десятки настроенных туннелей на абсолютно новый сервер за 1 минуту.

### Экспорт (Backup)
Создать копию можно двумя способами:

- Вызвать команду /backup в Telegram-боте (рекомендуется).

- Выбрать Пункт 9 -> 1) Создать локальную резервную копию в главном меню сервера (файл сохранится в /root/).

- Файл бэкапа содержит чистые метаданные: порты, протоколы, IP-адреса и конфигурации Watchdog.

### Импорт (Restore)
- Установите gokaskad на новый сервер базовой командой.

- Загрузите ваш файл бэкапа на новый сервер (например, в директорию /root/ через SFTP).

- В главном меню выберите Пункт 9 -> ```2) Восстановить инфраструктуру.```

- Укажите полный путь к файлу (например, ```/root/gokaskad_backup_2023-10-25.txt```).

    Алгоритм автоматически скомпилирует правила под сетевые интерфейсы нового сервера и перезапустит службы.

## 6. Механизм обновления (Self-Update)
Архитектура системы поддерживает бесшовное обновление без прерывания трафика. Если в репозитории kaskad-server на GitHub появилась новая версия:

- Откройте вашего Telegram-бота.

- Отправьте команду /update.

- Служба атомарно скачает новый файл, проверит его целостность и перезапустит системного демона. Текущие сетевые каскады при этом не разрываются.
